Uploading!!!

You made it!

Welcome to the final step. Now, there are two main methods of uploading, or flashing, firmware. The first is the one we'll be focusing on, the Command Line Interface (CLI). If you have an RP2040, however, you can use the Drag-and-Drop method instead if you'd like.

No matter which method you use, you must first put your microcontroller into bootloader mode. The first thing you need to do is plug in your keyboard while holding down the boot/reset button.

This is usually a small black button on the microcontroller, but you may need to temporarily short two pads.
Alternatively, if bootmagic is supported on your keyboard, (if bootmagic was set to true in the Compile step,) holding down the key at (0,0) on the matrix will put the keyboard into bootloader mode.

However you do it, hold the button/connection for a few seconds as you plug it in.

Command Line Interface (CLI)

Command Line Interface is an option supported by the vast majority of QMK-compatable bootloaders. After putting your device into bootloader mode, open QMK MSYS.

Inside, run one simple command: qmk flash -kb <keyboard> -km default, again replacing <keyboard> with the name of the main file you created before.

You're done! Skip to the bottom.

Drag-and-Drop (RP2040 only!)

The drag-and-drop method of flashing is simple and fast, but only completely supported by RP2040 processors. If you have an RP2040, this method is for you.

When you plugged in your bootloader, a drive file may have popped up. If it didn't, look in your file system for a drive named RPI-RP2 or something similar.

Inside this file, drag and drop the .uf2 compile file into it. It will automagically close, and your keyboard will be flashed! Congrats.

Farewell!

With that, your keyboard is done! I'm glad you used EasyQMK to code your custom keyboard. If you have any questions, comments, or suggestions, go to my other project's contact page to email me.

Well, that's it. Hope to see you again!
