OLED Setup
Uh oh! Looks like you didn't select OLED in the previous step! Please go back and select it if you'd like to use an OLED. If you aren't using it, keep on reading. ;)

Alright. Now we're getting into the good stuff. OLEDs are a more complicated feature, but with the right stuff we can do it. Firstly, here's a good video tutorial:

If you want to read a guide instead of the video, (video much better!! by scottokeebs!!!) read on.

Wiring

The OLED should be wired like so: (OLED -- > Microcontroller)

  1. SDA -- > SCK/SDA
  2. SCL -- > SCL
  3. VCC -- > VCC
  4. GND -- > GND

Stuff that you don't necessarily need to know because of the auto-code generator but you still can because it's right here

Code for OLED:

rules.mk 
OLED_ENABLE = yes
LTO_ENABLE = yes

This code first enables the OLED then enables LTO, or link time optimization. It helps run the OLED smoothly, but increases compile size and length.

There are two main types of OLEDs, 128x32's and 128x64's. The one on the left is the 128x32:

If you have the skinny one, you're all set. But if you have the thicc boi, you need one more line of code:

config.h 
#define OLED_DISPLAY_128x64

Finally, we're going to add stuff to keymap.c to get the OLED displaying stuff.

All your OLED code will go in a conditional block, like so in this example, which writes "Hello World!" to the OLED.

keymap.c 
#ifdef OLED_ENABLE
// Rotate OLED. THis rotates Clockwise, so if your OLED is upside down, put 180. If it needs to rotate counterclockwise 90 degrees, put 270.
oled_rotation_t oled_init_user(oled_rotation_t rotation) {
	return OLED_ROTATION_90;
}
// Draw to OLED
bool oled_task_user() {
	// Set cursor position
	oled_set_cursor(0, 1);
		
	// Write text to OLED
	oled_write("Hello World!", false);
		
	return false;
}

#endif

Useful Code

While this is all good stuff, it's not necessarily helpful. OLEDs are supposed to be helpful interfaces for the keyboard. This next code is meant to be both fun as well as helpful.

Here we see an OLED displaying whether Caps Lock is on or not:

keymap.c 
#ifdef OLED_ENABLE

// Rotate OLED
oled_rotation_t oled_init_user(oled_rotation_t rotation) {
    return OLED_ROTATION_90; 
}

// Draw to OLED
bool oled_task_user() {
    // Set cursor position
    oled_set_cursor(0, 1);

    // Caps lock status, very simple. host_keyboard_led_state(); is a function pre-defined, which returns a boolean of whether Caps Lock is on or not.
    led_t led_state = host_keyboard_led_state();
    oled_write_P(led_state.caps_lock ? PSTR("Caps Lock On") : PSTR("Caps Lock Off"), false);
		// You can change "Caps Lock On" and "Caps Lock Off" to the strings of your liking.

    return false;
}

#endif

This block tells you which layer your keyboard is on. This guide doesn't cover layers, but if your keyboard has them, this is where the OLED would come in handy. For more info on layers, which are a valuable and great part of any keyboard, check out this guide.

keymap.c 


// These names don't matter. Just include the same number as you have layers, and in the same order.
enum layer_names {
	_MAC_DEFAULT,
	_MAC_CODE,
	_MAC_NUM,
	_MAC_FUNC
}


#ifdef OLED_ENABLE

 
// Rotate OLED
oled_rotation_t oled_init_user(oled_rotation_t rotation) {
    return OLED_ROTATION_90; 
}

// Draw to OLED
bool oled_task_user() {
    // Set cursor position
    oled_set_cursor(0, 1);

    // Switch on current active layer
    switch (get_highest_layer(layer_state)) {
        case _MAC_DEFAULT :
            oled_write("Main Layer", false);
            break;
        case _MAC_CODE :
            oled_write("Code Layer", false);
            break;
        case _MAC_NUM : 
            oled_write("Number Layer", false);
            break;
        case _MAC_FUNC :
            oled_write("Function Layer", false);
            break;
    }

    return false;
}

#endif
